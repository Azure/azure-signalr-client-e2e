# JavaScript — WebPubSub Chat Client E2E Test Harness

E2E tests for the Azure Web PubSub chat client SDK
([`@azure/web-pubsub-chat-client`](https://github.com/Azure/azure-webpubsub/tree/main/sdk/webpubsub-chat-client)),
integrated into the same repository that runs the SignalR client E2E tests.

Unlike the Java/Swift clients (which talk to the local .NET `IntegrationTestServer` and
Azure SignalR), the chat client talks to **Azure Web PubSub** through a small local
**negotiate server** (`server.mjs`) that issues client-access URLs for the `chat` hub.

## Layout

| File | Purpose |
|------|---------|
| `server.mjs` | Negotiate server (`GET /negotiate?userId=…`) backed by Azure Web PubSub |
| `prepare-tests.sh` | Copies the SDK's own integration tests from the `webpubsub` submodule and rewrites their SDK import to the package name |
| `tests/*.ts` | The chat client SDK's integration tests (generated by `prepare-tests.sh`; git-ignored) |
| `package.json` | Deps: `@azure/web-pubsub-chat-client`, `@azure/web-pubsub`, `express`, `tsx` |

## Tests are not forked

The integration test logic lives in the chat client SDK itself
(`webpubsub/sdk/webpubsub-chat-client/tests`). `prepare-tests.sh` copies those
tests verbatim and only rewrites the SDK import specifier
(`../src/chatClient.js` → `@azure/web-pubsub-chat-client`), so the exact same
assertions run against either the source-packed (dev) or npm (stable) SDK — no
duplicated test code to keep in sync.

## SDK version source

- **Dev** — built from the `webpubsub` submodule (`Azure/azure-webpubsub`,
  `sdk/webpubsub-chat-client`), packed with `npm pack`, and installed into this harness.
- **Stable** — the published `@azure/web-pubsub-chat-client` from npm.

The tests import the package by name (`@azure/web-pubsub-chat-client`), so the same test
code runs against either source; only the installed version differs.

## Prerequisites

- Node.js 20+ (for the built-in test runner used via `tsx`).
- An **Azure Web PubSub** resource with:
  - Persistent Storage configured (Storage Account with Table enabled).
  - A `chat` hub with the Chat feature enabled, using that Persistent Storage.

## Running locally

```bash
cd javascript/chatclient
bash prepare-tests.sh   # copy SDK tests from the webpubsub submodule
npm install

# Terminal 1 — negotiate server
export E2E_WEBPUBSUB_CHAT_CONNECTION_STRING="<your-web-pubsub-connection-string>"
npm run start-server

# Terminal 2 — tests
npm test
```

Or use the repo-level `./run-all-tests.sh` / `./run-from-artifacts.sh`, which start the
negotiate server and run this suite alongside the other clients.
