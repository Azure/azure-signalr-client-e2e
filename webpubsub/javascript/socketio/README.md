# JavaScript — WebPubSub Socket.IO Extension E2E Test Harness

E2E tests for the Azure Web PubSub Socket.IO extension SDK
([`@azure/web-pubsub-socket.io`](https://github.com/Azure/azure-webpubsub/tree/main/sdk/webpubsub-socketio-extension)),
integrated into the same repository that runs the SignalR client E2E tests.

Unlike the Java/Swift clients (which talk to the local .NET `IntegrationTestServer` and
Azure SignalR), this suite targets **Azure Web PubSub** directly. The SDK's own mocha
tests spin up Socket.IO servers backed by Web PubSub — no separate negotiate server is
required.

## Layout

| File | Purpose |
|------|---------|
| `prepare-tests.sh` | Copies the SDK's own `test/` tree from the `webpubsub` submodule and rewrites its internal source imports (`../src`, `../src/common/utils`) to the package name |
| `tests/**/*.ts` | The Socket.IO extension SDK's test suite (generated by `prepare-tests.sh`; git-ignored) |
| `package.json` | Deps: `@azure/web-pubsub-socket.io`, `socket.io`; test tooling: `mocha`, `ts-node`, `socket.io-client`, `expect.js`, `supertest` |
| `tsconfig.json` | `ts-node` transpile-only config used by the mocha runner |

## Tests are not forked

The test logic lives in the Socket.IO extension SDK itself
(`webpubsub/sdk/webpubsub-socketio-extension/test`). `prepare-tests.sh` copies those
tests verbatim and only rewrites the internal import specifiers:

- `../src` (public entry, re-exports `useAzureSocketIO` / `negotiate` / `NegotiateOptions`) → `@azure/web-pubsub-socket.io`
- `../src/common/utils` (`debugModule`, an internal `debug` wrapper) → an inline `debug` shim

so the exact same assertions run against either the source-packed (dev) or npm (stable)
SDK — no duplicated test code to keep in sync.

## SDK version source

- **Dev** — built from the `webpubsub` submodule (`Azure/azure-webpubsub`,
  `sdk/webpubsub-socketio-extension`, which also builds its `sdk/server-proxies`
  dependency), packed with `npm pack`, and installed into this harness.
- **Stable** — the published `@azure/web-pubsub-socket.io` from npm.

The tests import the package by name (`@azure/web-pubsub-socket.io`), so the same test
code runs against either source; only the installed version differs.

## Prerequisites

- Node.js 20+.
- An **Azure Web PubSub** resource with an `eio_hub` hub configured (the hub name the
  SDK test suite uses).

## Running locally

```bash
cd webpubsub/javascript/socketio
bash prepare-tests.sh   # copy SDK tests from the webpubsub submodule
npm install

export WebPubSubConnectionString="<your-web-pubsub-connection-string>"
export WebPubSubHub="eio_hub"
export SocketIoPort="3000"
npm test
```

Or use the repo-level `./run-all-tests.sh` / `./run-from-artifacts.sh`, which set the
above from `E2E_WEBPUBSUB_SOCKETIO_CONNECTION_STRING` and run this suite alongside the
other clients.
